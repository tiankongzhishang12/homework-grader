# Handoff Update Checklist

## Before Updating

- Identify whether the user asked for a snapshot, decision record, or index update.
- Check current code facts before recording technical stack, routes, APIs, or tables.
- Keep handoff focused on current state and next collaborator needs.

## Include

- current phase;
- completed capabilities;
- active risks;
- document-code conflicts;
- next steps;
- recent changes and verification.

## Avoid

- dumping full architecture docs into a handoff;
- duplicating README;
- claiming planned work is already complete;
- deleting or moving scripts;
- rewriting root `SKILL.md` as old platform Skill instructions.

## After Updating

- Report files changed.
- Mention any source-of-truth checks performed.
- Mention unresolved risks.
