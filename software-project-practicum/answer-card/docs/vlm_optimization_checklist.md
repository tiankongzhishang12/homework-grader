# VLM 优化清单

## 1. 文档目标

本文档用于整理当前答题卡批改项目在 VLM/大模型视觉识别方向上的优化重点。

当前项目尚未单独接入传统 OCR 引擎，主要依赖：

- VLM 直接看图识别文字、结构和涂卡结果
- 程序规则进行答案匹配、判分和人工复核触发

因此，VLM 优化的核心不是“让模型更自由发挥”，而是：

- 把输入切小
- 把任务变窄
- 把输出收紧
- 把最终判分尽量交给程序规则


## 2. 总体原则

- 让 VLM 只做“感知”和“提取”，尽量不直接做最终判分决策。
- 输入尽量按题、按空、按区域切小，不让模型整页混看。
- 每类题型使用专属 prompt，不共享一个通用大 prompt。
- 能程序判分的题型，优先走“识别结果 + 程序判分”链路。
- 不确定时优先返回 `null` 或触发人工复核，不强行猜测。
- 每次优化都必须用验证样本回放，不凭主观感觉判断效果。


## 3. 客观题优化

### 3.1 单选题

当前方向：

- 作为标准涂卡题处理
- VLM 只识别学生填涂的选项
- 程序负责与 `standard_answers.yaml` 比对并计算得分

优化重点：

- 只输入单选题区域，不要与判断题或填空题混看
- 输出固定为 `A/B/C/D/null`
- 增加多视图输入：
  - 原始裁图
  - 红色增强图
  - 局部放大图
- 如发现多选、擦改或无法确认，直接返回 `null`


### 3.2 判断题

判断题本质上也是涂卡题，不应按自然语言判断题处理，而应按“二选一涂卡题”处理。

推荐处理方式：

- 左槽位对应 `T`
- 右槽位对应 `F`
- 优先通过程序规则比较左右槽位填涂强度
- VLM 仅作为辅助手段，不作为主判定器

优化重点：

- 逐题小块裁图，不再整块依赖 VLM 判断
- 必要时保留：
  - 整块识别
  - 分行识别
  - 逐题识别
  作为调试手段
- 正式主链路优先使用“槽位强度比较 + 程序判分”
- 左右差异太小、整体太浅时返回 `null`

当前阶段目标：

- 先将判断题做到接近无差错
- 明显减少“整列全判成 T/F”的系统性误差


## 4. 填空题优化

### 4.1 处理原则

填空题 `21-25` 的答案术语相对固定，属于“短词识别 + 候选匹配”问题，不适合完全开放式生成。

推荐链路：

1. 逐空裁图
2. VLM 只提取该空的短词文本
3. 程序进行归一化与候选匹配
4. 必要时再做小候选集兜底判断


### 4.2 优化重点

- `21-25` 每空单独裁图，不再整块一起识别
- VLM 输出仅包含：
  - `text`
  - `evidence`
- 不允许模型直接输出“标准答案”
- 每题只在本题候选答案集内做匹配

程序后处理建议：

- 去空格
- 去标点
- 全半角统一
- 常见近形字、轻度错写容错
- alias 匹配

重点难点：

- `22` 题这类短词最容易误识别
- 应为每题建立独立候选集，不要跨题乱匹配

当前阶段目标：

- 优先解决“学生写对了但系统识别失败”的问题
- 尽量让填空题达到接近无差错


## 5. 主观题优化

### 5.1 文本主观题 26-30

推荐链路：

- 每题单独裁图
- VLM 先提取答案要点或转写内容
- 程序或 rubric 规则完成评分

优化重点：

- 不让模型同时做“识别 + 理解 + 总分决策”
- 输出固定为：
  - 提取到的要点
  - 匹配到的评分点
  - 题目得分
  - 证据


### 5.2 图示题 31-33

推荐链路：

- 每题单独大块裁图
- 不与其它题混看
- VLM 优先做结构识别，再映射到 rubric

优化重点：

- 识别参与者、实体、过程、关系、连线、数据流等结构
- 不直接整页给总分
- 对低清晰度、结构不完整、字迹过密情况触发人工复核


## 6. Prompt 设计原则

每类题型都应使用专属 prompt，至少区分为：

- 单选题 prompt
- 判断题 prompt
- 填空题 prompt
- 文本主观题 prompt
- 图示题 prompt

共同要求：

- 输出格式固定
- 不允许自由发挥
- 明确“不确定返回 null”
- 明确“不得参考教师批注”
- 明确“只识别当前题/当前空/当前区域”


## 7. 输入增强策略

VLM 输入不应只依赖一张原图，应按题型组合更适合识别的视图：

- 原始裁图
- 放大图
- 红色增强图
- 灰度图
- 二值化增强图

适用建议：

- 涂卡题优先考虑原图 + 放大图 + 局部增强
- 短词填空优先考虑单空放大图 + 灰度/增强图
- 图示题优先考虑原图大块 + 局部放大图


## 8. 输出约束

### 8.1 客观题

- 单选题只允许 `A/B/C/D/null`
- 判断题只允许 `T/F/null`

### 8.2 填空题

- 只允许输出：
  - `text`
  - `evidence`
- 不允许直接给“判对/判错”

### 8.3 主观题

- 只允许输出：
  - `score`
  - `matched_points`
  - `evidence`

统一要求：

- 支持 `null`
- 不确定时不要硬猜


## 9. 人工复核策略

以下情况建议直接触发人工复核：

- 客观题出现多题 `null`
- 判断题左右槽位强度差异过小
- 填空题未命中本题候选集
- 图示题结构识别不完整
- 主观题 evidence 过短
- 题目答案存在明显图像模糊或遮挡


## 10. 验证指标

每次进行 VLM 优化后，应至少统计以下指标：

- 单选题命中率
- 判断题命中率
- 填空题满分率
- 每题 `null` 比例
- 总分 MAE
- 各题 MAE
- 误差最大的样卷清单

验证时要重点关注：

- 是否减少了“系统性整列误判”
- 是否减少了“学生写对但系统给 0 分”
- 是否降低了人工复核比例


## 11. 当前优先级

建议按以下顺序推进：

1. 判断题稳定到接近无差错
2. 填空题稳定到接近无差错
3. 再优化主观题中误差最大的题：
   - `26`
   - `30`
   - `31`
   - `32`
   - `33`
4. 最后再跑 `30` 份验证集全量验证


## 12. 当前阶段结论

现阶段最值得优先投入优化的部分不是全部题型同时推进，而是：

- 判断题：
  按涂卡题处理，尽快稳定程序规则
- 填空题：
  按“逐空识别 + 候选匹配”处理，尽快减少识别漏判

当这两类“答案较死”的题先稳定下来后，再集中处理主观题误差，整体收益会更高。
