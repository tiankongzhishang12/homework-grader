# Codex Project Context: 智能阅卷系统

本文档是 Codex 在本仓库中协作时使用的项目上下文，不再是旧 Skill 安装说明。它的作用是帮助协作者理解业务边界、目录职责、工作流程和修改约束。

## 项目定位

本项目是一个智能阅卷系统，面向课程作业、软件项目实训报告、考试答题卡和类似教学评价场景。系统目标不是只让模型给出一个分数，而是支撑可配置、可追踪、可复核、可导出的完整阅卷流程。

当前项目默认使用 ChatGPT / OpenAI-compatible 模型能力，并通过 Codex 辅助开发、整理文档和维护代码。历史上保留下来的 Rubric、Prompt、Python 批处理脚本和质量控制材料仍可继续使用，但项目不再以旧 Skill 体系作为主要定位。

## Codex 协作原则

- 保持当前智能阅卷工作流稳定，除非用户明确要求重构。
- 不要因为看到 Python 脚本就判断它们是废弃内容；部分脚本仍被后端或当前演示流程使用。
- 清理文档时优先去除旧助手平台专属叙事，而不是删除评分方法、Rubric、批处理或质量控制能力。
- 修改后端接口前，先检查前端调用、数据库表和脚本调度关系。
- 修改前端页面时，保持面向教师的工作台式体验，避免营销页或无关装饰。
- 涉及成绩、评分、导出和教师复核的功能，应优先保证可审计和可回退。

## 核心业务流程

1. 组织和课程初始化：维护教师、学生、课程、教学班和任务上下文。
2. 阅卷任务配置：配置任务范围、提交要求、标准答案、Rubric、题目区块和导出模板。
3. 学生提交处理：上传或导入学生文件，生成可追踪的提交记录。
4. 批量阅卷：系统启动阅卷任务，执行解析、评分、聚合和质量标记。
5. 教师复核：查看分项得分、证据、扣分点、置信度和异常标记。
6. 成绩确认：教师确认或调整最终成绩。
7. 分析和导出：生成班级统计、学生明细、导出文件和发布记录。

## 后端注意事项

关键类：

- `GradingWorkflowService`：阅卷任务启动、进度和教师确认/调整。
- `PythonScriptClient`：当前用于调度 Python 预处理、评分和导出脚本。
- `ExportController`：成绩导出和最新报表下载。
- `FrontendViewController`：为前端原型聚合展示数据。
- `CrudJdbcRepository`：通用 JDBC 数据访问封装。

当前 `application.yml` 中的 `grader.python.*` 配置仍然有效。除非用户要求替换为纯 Java 工作流，否则不要移除这些配置或相关类。

## 前端注意事项

前端位于 `software-project-practicum/frontend-prototype/`，使用 Vue 3、TypeScript、Vite 和 Pinia。

设计和交互应服务于教师的高频工作：

- 页面第一屏应直接进入任务、配置、阅卷、分析或导出功能。
- 状态、分数、风险、置信度和复核动作要清楚。
- 配置页应帮助教师逐步完成任务准备，而不是展示抽象宣传文案。
- 导出和发布功能要让教师知道当前数据来源和可用状态。

## Python 脚本定位

Python 脚本是系统辅助工具链，主要用于：

- 文档预处理和格式转换。
- 批量评分和模型调用。
- 答题卡识别、标准答案推断和验证评分。
- 统计分析、校准和 Excel 导出。

这些脚本可以使用 OpenAI API 或 OpenAI-compatible 网关。它们不是特定助手平台绑定物，清理旧叙事时不应顺手删除脚本。

## Rubric 和评分原则

Rubric 是评分标准的结构化表达，通常包含：

- 评分维度或题目区块。
- 权重或满分。
- 判分依据、标准答案或锚点描述。
- 门禁检查、质量标记和评语要求。

评分结果应尽量包含：

- 分项得分。
- 证据或命中依据。
- 扣分原因。
- 置信度。
- 教师复核建议。

最终成绩应以教师确认后的 `final_result` 为准。

## 质量控制原则

- 低置信度结果应进入复核。
- 解析失败、文件缺失、格式异常和模型输出异常应保留错误记录。
- 批量阅卷后应检查分数分布、异常高低分和题目区块失分集中情况。
- 评分规则调整后，应保留版本和调整原因。
- 学生身份信息和模型输入应尽量分离，避免无必要暴露。

## 文档维护方向

当前文档维护目标是“去旧平台叙事，但不破坏工作流”：

- 删除旧 Skill 安装方式。
- 删除特定供应商或模型平台的专属推荐。
- 将平台专属视觉模型表述改为多模态模型或视觉模型。
- 将 “Skill 激活” 改为 “项目上下文”。
- 保留 Rubric、质量控制、批量处理、导出和 Python 辅助脚本说明。
